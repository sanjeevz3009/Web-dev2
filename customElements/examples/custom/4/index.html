Lifecycle Callback methods

This clock at the end of this page updates every second, but it only updates because it's currently connected to this document. It's good practice not to be doing unnecessary work, and this clock knows when it is connected and disconnected thanks to two standard functions that we have implemented. They are connectedCallback and disconnectedCallback. The browser calls these functions for us whenever our element is added to or removed from a document.

To see this working, inspect this page and paste the following into the console: 

const clock = document.querySelector("digital-clock");
clock.remove();
clock;

Open the make element up in the console and you'll see the time that it was removed from the document. If you re-add it to the document using the following code… 

document.body.append(clock);

&hellip then the time will briefly appear as it was before it was removed from the page (and then the new interval timer will kick-in and update it to the correct time.
